# GUILD AI 業務フロー図

出品 → AI 鑑定 → 登記 → 拡散 → 購入 → 信用反映 の循環を可視化する。

---

## 1. 全体循環フロー（flowchart）

```mermaid
flowchart TD
    A[個人オーナー\n出品開始] --> B[/app/sell\nGitHub URL 入力]
    B --> C[CCAF アップロード\nor 自動生成]
    C --> D[ai-auditor\n思考密度+Vercel稼働解析]
    D -->|S/A/B 判定| E[ランク確定]
    E --> F[Floor Price 自動算出]
    F --> G[登記送信\n知能資産 DB]
    G --> H[Marketplace 公開]
    H --> I[Ambassador が拡散\nDiscord/X]
    I --> J[discord-bridge\nアクション計測]
    J --> K[Trust Score 再計算]
    K --> L{toA / toB / 個人\n購入}
    L --> M[売上配分\nオーナー + Ambassador]
    M --> N[品質履歴 更新]
    N --> K
    K --> F

    classDef soul fill:#E8C46A,stroke:#0F0F12,color:#0F0F12;
    class E,G soul;
```

---

## 2. 出品 → 鑑定 → 登記（sequenceDiagram）

```mermaid
sequenceDiagram
    autonumber
    participant U as 個人オーナー
    participant S as /app/sell
    participant GH as GitHub
    participant AA as ai-auditor
    participant DB as 知能資産DB

    U->>S: GitHub URL 入力
    S->>GH: リポジトリ検証
    GH-->>S: メタ情報
    U->>S: CCAF JSON アップロード
    S->>AA: audit(ccaf, vercelUptimeDays)
    AA-->>S: { rank: S/A/B, score, reasons }
    S-->>U: AI 鑑定プレビュー
    U->>S: basePrice 設定 → 登記送信
    S->>DB: Listing 保存（rank, floorPrice）
    DB-->>S: listingId
    S-->>U: 登記完了
```

---

## 3. 拡散 → Trust Score 反映（sequenceDiagram）

```mermaid
sequenceDiagram
    autonumber
    participant Amb as Ambassador
    participant DC as Discord
    participant DB2 as discord-bridge
    participant TS as TrustScore Service
    participant UI as TrustScore UI

    Amb->>DC: share / endorse / react / bug-report
    DC->>DB2: webhook event
    DB2->>DB2: レート制限（1日50pt）+重み計算
    DB2->>TS: discordContribution += weighted
    TS->>TS: score = 0.5*Q + 0.3*D + 0.2*X
    TS-->>UI: スコア & ランク更新
    TS-->>Amb: 売上シェア計算へ反映
```

---

## 4. 購入 → 信用反映ループ（flowchart）

```mermaid
flowchart LR
    P[購入完了] --> Q[評価フィードバック]
    Q --> R[qualityHistory 更新]
    R --> S[TrustScore 再計算]
    S --> T[Floor Price 再算出]
    T --> U[露出ロジック更新]
    U --> P
```

## 5. 状態遷移（stateDiagram-v2）

```mermaid
stateDiagram-v2
    [*] --> Drafting: 出品開始

    Auditing --> RankedB: 標準判定

    Amplified --> Sold: 購入
```
