## 3. 仕様整合性チェック（doc ⇄ code）

### ランク閾値（ai-auditor）
- 用語集: S=density≥80 ∧ uptime≥30 ∧ intent。A=density≥60 ∧ uptime≥7。B=その他。
- マスター設計図 §3: 同上。
- `src/lib/ai-auditor/index.ts`: 同上の閾値（`S_THRESHOLD={80,30}`, `A_THRESHOLD={60,7}`, intent ガード）。
- ✓ 一致

### Trust Score の重み
- 用語集 / マスター設計図 §4: `0.5*qualityHistory + 0.3*discordContribution + 0.2*xAmplification` → 0–100 → ×10 で 0–1000。
- `src/lib/trust-score/index.ts`: 同式。
- ✓ 一致

### Floor Price
- マスター設計図 §4: `floorPrice = basePrice × (1 + score/2000)`、最大 +50%。
- `src/lib/ai-auditor/index.ts#computeFloorPrice`: `1 + min(score,1000)/2000`。
- ✓ 一致

### Discord 重み・日次キャップ
- マスター設計図 §5 / 運用マニュアル §6: share=5, endorse=3, react=1, bug-report=4、1 日 50pt。
- `src/lib/discord-bridge/index.ts`: 同値（`DISCORD_WEIGHTS`, `DAILY_CAP=50`）。
- ✓ 一致

### sell ページのステップ
- マスター設計図 §6: GitHub URL → CCAF → AI 鑑定 → 価格 → 登記。
- `src/app/sell/page.tsx`: Step 1〜4 + 登記送信。
- ✓ 一致

### 用語表記
- 用語集の主要用語（GUILD AI / Trust Score / CCAF / Proof of Thought / Ambassador / Floor Price / 魂の登記 / 黒衣 / 柿 / toA / toB）が
  全資料・コードコメントで統一されていることを確認。
